# 成员 C 故事线 D3 图形设计方案

章节：同行 · 巨人的肩膀与同行者

目标：用 3 张主图把“诺奖颁给个人，但科学由共同体生成、传递与放大”这条叙事讲清楚。整体设计强调可读性优先，避免一次性展示过密网络。

## 1. 数据概况与设计判断

当前已整理出的中间数据量足以支撑三张图，但不同图的复杂度应区别处理：

- 图 1 对应 874 条 prize-winning 论文记录，适合做分年代的分布图。
- 图 2 对应 2320 个节点、2400 条边，适合做可过滤的合作网络，不适合默认全量展开。
- 图 3 对应 544 个节点、10512 条内部引用边，密度很高，必须依赖边权过滤或分学科切片。

基于这个数据量，建议的设计原则是：

1. 先让观众理解“团队规模在变大”。
2. 再让观众看到“得主背后有稳定合作者和桥梁作者”。
3. 最后让观众看到“得主之间存在可追踪的引用传承”。

## 2. 图 1：团队不再是配角——获奖论文团队规模如何演变

### 2.1 叙事目标

回答一个最基础的问题：诺奖相关论文的作者数量，是否随着年代系统性上升？如果是，那它说明诺奖成果越来越依赖团队协作，而不是少数孤立天才的瞬间爆发。

### 2.2 使用数据

数据文件：

- figure1_team_size_prize_papers.csv

关键字段：

- category
- laureate_name
- prize_year
- decade
- author_count
- is_prize_winning_paper

### 2.3 推荐图形

推荐使用“分年代的 boxplot + jitter/beeswarm 组合图”。

原因如下：

- boxplot 能快速看出中位数、四分位和离群值。
- jitter 或 beeswarm 能保留每篇论文的离散性，避免 boxplot 过于抽象。
- 这个图天然适合展示时间趋势和跨学科比较。

### 2.4 视觉编码

- x 轴：Prize year 或 decade，推荐按 decade 分组。
- y 轴：author_count。
- 颜色：学科类别，Physics / Chemistry / Medicine。
- 点形：可统一为圆点，避免视觉噪音。
- 透明度：0.35 到 0.55，防止堆叠过密。

### 2.5 默认展示逻辑

建议默认展示全部 prize-winning 论文，但按 decade 聚合；如果一个 decade 内点太多，可切换到“boxplot 为主，点只在 hover 时突出”的模式。

建议在图旁边放一句结论文本：

“诺奖不是一个人的灵光一现，而是被时代推动的团队协作方式。”

### 2.6 需要注意的细节

- Physics、Medicine 的 openalex_authors 需要按分号拆分计数。
- Chemistry 已经有 openalex_authors_count，可直接使用。
- 如果未来要比较“论文发表年”和“获奖年”，建议只在 tooltip 中保留 pub_year，不作为主轴。
- 对极端大团队可做上限提示，例如在 tooltip 中说明“100+ 作者”。

### 2.7 D3 实现建议

- 用 d3.boxplot 或自定义 boxplot 组件。
- beeswarm 可用 force simulation 做轻度排布。
- tooltip 显示：学科、得主、论文题目、发表年、作者数、是否 prize-winning。
- 若要更强调趋势，可在 boxplot 上叠加每个学科的 median line。

## 3. 图 2：同行者矩阵图——谁和谁最常一起出现

### 3.1 叙事目标

回答两个问题：

- 哪些同行者会在多个得主身边反复出现？
- 哪些得主背后存在稳定、可识别的合作骨架？

这一图不再展示毛球式网络，也不再依赖复杂弧线，而是把“共同体的骨架”压缩成一个更易读的矩阵。

### 3.2 使用数据

数据文件：

- figure2_coauthor_nodes.csv
- figure2_coauthor_edges.csv

关键字段：

- node_name
- node_type
- category
- paper_count
- dominant_category
- source
- target
- weight

### 3.3 数据现实判断

当前网络总体规模为 2320 个节点、2400 条边。这个量级说明网络是可画的，但不能默认全展开，否则会出现以下问题：

- 得主节点会被大量低频作者淹没。
- 合作者节点之间的重复程度不够高，容易形成“毛球状”网络。
- 观众无法识别桥梁节点。

因此，图 2 必须做筛选和层次化显示。

### 3.4 推荐图形

推荐使用“得主 × 桥梁作者矩阵热图”。这是最清楚的表达方式，也最符合 D3 的交互优势：排序、缩放、悬停联动都很自然。

推荐结构：

- 行：按累计合作强度或获奖年份排序的代表性得主。
- 列：按连接得主数量排序的桥梁作者。
- 单元格：颜色越深，表示该对人物在 prize-winning papers 中的合作越频繁。

默认只展示经过筛选后的前若干行和前若干列，保证图面不会过宽。

### 3.5 建议筛选规则

默认只显示以下内容：

1. 边权 weight 小于 2 的连接默认隐藏。
2. 只保留连接多位得主的桥梁作者。
3. 只展示前若干位代表性得主和前若干位桥梁作者，确保版面可读。

这个策略和当前数据量匹配，因为我们已经观察到有大量作者会重复出现在不同得主身边，说明“桥梁作者”是稳定存在的。

### 3.6 交互建议

- hover 单元格时，高亮对应整行和整列。
- hover 行标签时，高亮该得主对应的所有合作对象。
- hover 列标签时，高亮该桥梁作者连接到的所有得主。
- 提供“桥梁作者”开关，打开时只保留连接多位得主的作者。

### 3.7 结论文本建议

“诺奖共同体不是松散的偶遇，而是由少数长期搭档和跨得主的桥梁作者反复编织出来的。”

### 3.8 D3 实现建议

- 使用 band scale 做矩阵布局，避免网络图的路径交叉。
- 用 sequential color scale 表达合作强度，保留清晰的读数路径。
- hover 时做行列联动高亮，让单元格既能读“点”，也能读“关系”。
- 如果版面允许，可以在右侧加一个轻量的颜色图例，而不是再增加第二层图形。

## 4. 图 3：巨人的肩膀在共同体内部——得主之间的互引/致敬网络

### 4.1 叙事目标

回答“诺奖共同体是否真正互相引用”的问题。这里不是简单看引用数量，而是要把“传承链”和“致敬网”讲出来。

### 4.2 使用数据

数据文件：

- figure3_internal_citation_nodes.csv
- figure3_internal_citation_edges.csv

关键字段：

- node_name
- category
- prize_year
- source
- target
- weight

### 4.3 数据现实判断

当前内部引用网络有 544 个节点、10512 条边。边数远大于节点数，说明这不是稀疏关系图，而是一个高度纠缠的知识传承网络。

已知统计显示：

- 6537 对得主间引用边权至少为 2。
- 3174 对边权至少为 5。
- 1665 对边权至少为 10。
- 最大边权达到 3152。

这意味着如果不裁切，网络会完全不可读。图 3 必须默认做强过滤。

### 4.4 推荐图形

推荐使用“弧线图 arc diagram”，优先于普通力导向图。

原因如下：

- 得主可以按照 Prize year 排列，时间顺序清晰。
- 引用方向可以通过弧线和箭头表达。
- 叙事感更强，更像“从前辈到后辈”的传承路径。
- 比力导向图更适合讲故事，而不是展示纯网络拓扑。

### 4.5 排序方式

建议横轴按照以下规则排序：

1. 先按 Prize year 升序。
2. 同一年内按 category 排序。
3. 再按 node_name 排序。

如果想强调学科内部的传承，也可以提供一个切换：按 category 分面后，在各自面板内按 Prize year 排序。

### 4.6 默认筛选规则

建议默认只展示 weight >= 5 的内部引用边。

如果画面仍然拥挤，可改为：

- 默认展示 weight >= 10。
- hover 某位得主时，再展开其全部相关边。

这样既能保证图面清洁，又不会丢失高频传承关系。

### 4.7 交互建议

- hover 某节点时，高亮其所有出边和入边。
- 点击某节点时，只展示与该节点相关的前后若干年邻居，形成局部故事段。
- 支持按学科筛选，便于比较不同领域的内部引用文化。

### 4.8 结论文本建议

“诺奖共同体并非互不相干：引用把他们编织成传承链与致敬网。”

### 4.9 D3 实现建议

- 使用 arc path 生成弧线，边的曲率可固定，避免图形过于复杂。
- 节点半径按引用总量或年份统一，不建议节点大小过多映射变量，以免破坏故事节奏。
- 边颜色可按来源学科或引用方向统一编码，建议引用方向更重要，颜色只做轻提示。

## 5. 如果需要第 4 张图

如果版面必须有 4 张图，建议不是再做一个重网络图，而是做“学科对比总结图”。

推荐形式：

- 三学科的团队规模中位数对比条形图，或者
- 三学科的“桥梁作者数量”对比小图，或者
- 三学科的“内部引用边权分布”对比图。

这个第 4 张图的作用是收束结论，而不是新增信息层。

## 6. 统一的视觉与交互规范

### 6.1 颜色建议

- Physics：冷色系，偏蓝。
- Chemistry：中性暖色系，偏橙。
- Medicine：偏红或酒红，但饱和度不要太高。

如果要给合作者和得主区分，建议：

- 得主用高饱和主色。
- 合作者用灰阶或低饱和同色系。

### 6.2 字体与标签

- 图标题必须是故事句，不只是图表名。
- 标签尽量短，长姓名只在 tooltip 里完整显示。
- 对密集网络，默认不显示全部标签，只显示关键节点。

### 6.3 统一交互

- hover 统一使用高亮邻接、淡化非邻接。
- click 统一使用“锁定视图”模式。
- tooltip 统一展示：类别、年份、权重、姓名、必要时显示论文标题。

### 6.4 响应式与性能

- 网络图建议支持最小宽度降级，不要在移动端硬撑全量布局。
- 图 2、图 3 一定要预先聚合数据，不要在浏览器端做全量字符串解析。
- 边权大于阈值后再渲染，可明显降低初次加载压力。

## 7. 推荐的最终落地顺序

1. 先完成图 1，因为它最稳定、最不容易失败。
2. 再完成图 3，因为内部引用网络的叙事性最强，但需要较严格过滤。
3. 最后完成图 2，因为合作网络最容易出现视觉拥挤，需要根据前两图的版式反推过滤强度。
